System Overview

Purpose

The object of this project is move the existing VCP v2 functionality and content from Drupal 7 into the Python Django framework. The site will maintain its existing functionality with the benefit of being easier to maintain and upgrade within its new framework habitat. This framework move will allow us to clean up existing functionality provided by Drupal that is unnecessary within the context of application as well as implement additional features as requested by the client.

Scope

Setup external media server for hosting & converting video files. Move existing application functionality from Drupal CMS to Django framework. Migrate existing content from current VCP sites into new application.

  • dard programming/web programming, database programming, web development and project maintenance - $50/hr.

ARC pricing is intended to cover costs, while offering PSU researchers and associated agencies excellent value for professional development and programming services. In addition, we offer valuable professional experience for undergraduate and graduate student programmers.

Scope of work and cost

Based on the agreed upon scope of work, all work will be performed to completion at the cost estimates listed here. Changes to the original scope of work

Changes to the original scope of work, once decided upon, will result in one or all of the following: additional cost, lengthened development time, and reduction of deliverables if budget constraints are an issue.

Bug fixes

All bug fixes are addressed with no further charge, EXCEPT for:

  • Browser support due to patches, version upgrades, and new browser releases.
  • General bug fixes for work done by other contractors or site owners.

This work will be charged on a time and materials basis at the standard rates listed above.

Ongoing maintenance

Web applications require ongoing support; browsers are constantly being upgraded and adding new technologies. This can cause problems with existing websites, typically with JavaScript, CSS, and plug-in compatibility. In addition, upgrades on web servers and in programming languages can cause issues as well. Our clients may choose to handle the necessary site modifications themselves, hire an independent contractor or staff (student), or contract with ARC to provide this service.

ARC can implement and bill for these upgrades either on a time and materials basis, or these can be addressed as part of a budgeted, ongoing maintenance plan in a Service Level Agreement (SLA). The terms of this maintenance plan will be based on the scope and complexity of a given project. The SLA should be arranged prior to the completion of a project.

Typical maintenance addresses:

  • Browser support due to patches, version upgrades, and new browser releases.
  • General bug fixes.
  • Changes required due to normal system upgrades because of performance or security.

This does not account for new features or ongoing development. Fixes for work done by other contractors or site owners may incur higher rates.
